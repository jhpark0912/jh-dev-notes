# Day 020. GitHub Copilot은 내 코드를 어떻게 아는 걸까? — AI 코딩 도구와 코드 보안

> 🗓️ 2026-04-01

## 🤔 질문

GitHub Copilot은 로컬에서 학습하는 건가? 실제 동작 구조는 어떻게 되어 있나?  
내 코드가 클라우드로 전송되면 유출 위험은 없는가?  
기업들은 AI 코딩 도구의 보안 문제에 어떻게 대처하고 있나?

---

## 📌 핵심 요약

Copilot은 **로컬에서 학습하지 않으며**, 매 요청마다 현재 파일·열린 탭·프로젝트 구조를 컨텍스트로 수집하여 **클라우드 LLM에 전송**하고 응답을 받는 구조다.  
2026년 4월 24일부터 개인 플랜(Free/Pro/Pro+)은 **코드 스니펫이 AI 학습에 기본 사용**되며, Business/Enterprise만 학습 미사용이 보장된다.  
기업은 **기업용 플랜, 자체 호스팅(Tabnine/Tabby), 전면 금지, 사용 가이드라인** 중 보안 요구사항에 맞는 방식을 선택하고 있다.

---

## 🔍 Part 1. 흔한 오해 — "로컬에서 학습한다"

Copilot이 내 코드를 로컬에서 학습해서 추천하는 것이 아니다. 클라우드에 있는 LLM에 요청을 보내는 구조다.

### 1-1. 실제 아키텍처 — 3개의 계층

```
1. 로컬 (IDE): 컨텍스트 수집 + 프롬프트 구성 + 응답 표시
2. 클라우드 (Microsoft Azure): LLM이 코드 생성 + 보안 필터링
3. 외부 데이터: AI 모델에 정보 제공

→ 로컬에서는 "수집과 표시"만
→ 실제 AI 추론은 전부 클라우드에서 발생
→ 인터넷 끊기면 Copilot 동작 안 함
```

### 1-2. "내 코드를 아는 것 같은" 이유 — 컨텍스트 수집

Copilot이 매 요청마다 수집하는 것: 현재 편집 중인 파일, 열려있는 다른 탭, 파일 경로와 이름, 프로젝트 구조, 주석과 함수명, import문. 이 정보를 프롬프트에 포함시켜 LLM에 전송하면, LLM이 컨텍스트를 보고 다음 코드를 예측한다.

"학습"처럼 느껴지는 이유는 **매번 주변 코드를 새로 읽어서 프롬프트에 포함**시키기 때문이다. IDE를 껐다 켜면 "기억"하는 건 없다.

### 1-3. 사전 학습(Training)과 추론(Inference)의 차이

**사전 학습:** GitHub 공개 저장소 수십억 줄의 코드로 이미 완료. OpenAI/Anthropic/Google 데이터센터에서 수개월·수억 달러가 드는 과정. 사용자 컴퓨터에서 절대 일어나지 않음.

**추론:** 타이핑할 때 실시간으로 발생. 컨텍스트 수집 → 클라우드 LLM 전송 → 응답 반환. 이것도 클라우드에서 처리.

**로컬에서 일어나는 것:** 컨텍스트 수집, 프롬프트 구성, 응답 표시. 이것만 로컬.

---

### 1-4. Copilot의 모델 진화

2021년 Codex(GPT-3 기반, 단일 모델)로 시작하여, 현재는 GPT-4o, Claude, Gemini 등 **여러 모델 중 선택 가능한 멀티 모델 플랫폼**으로 진화. 2025년에는 Agent Mode(자율 멀티파일 편집), 2026년에는 Coding Agent(이슈→PR 자동 생성)까지 확장.

### 1-5. 추천이 빠른 이유 — 엔지니어링

엣지 캐싱(자주 나오는 패턴 캐싱), 투기적 프리페치(다음 요청을 미리 생성), Diff 모델(변경 부분만 반환), 인라인 자동완성 전용 경량 모델 사용 등으로 빠른 응답 속도를 달성한다.

## 🔍 Part 2. 코드 보안 이슈 — 2026년 3월 정책 변경

### 2-1. 2026년 4월 24일부터 바뀌는 것

GitHub가 2026년 3월 25일 개인정보 처리방침을 업데이트. Copilot Free, Pro, Pro+ 사용자의 인터랙션 데이터(코드 스니펫, 프롬프트, 제안, 컨텍스트)를 AI 모델 학습에 **기본 사용(opt-out 방식)**으로 변경.

이전에는 사용자가 동의해야 수집(opt-in)했지만, 이제는 **직접 끄지 않으면 자동 수집**. Business/Enterprise는 영향 없음.

### 2-2. 플랜별 데이터 처리 차이

```
Free / Pro / Pro+ (개인):
  ❌ 코드 스니펫 AI 학습에 사용 (기본값!)
  ❌ 프롬프트/제안 28일 보관
  ❌ Microsoft 계열사와 공유 가능
  ✅ opt-out 설정으로 끌 수 있음

Business ($19/user/월):
  ✅ AI 학습 미사용
  ✅ IDE 접근 시 프롬프트/제안 미보관
  ✅ 조직 단위 정책 설정

Enterprise ($39/user/월):
  ✅ AI 학습 미사용
  ✅ 프라이빗 클라우드 옵션
  ✅ IP 보호 필터
```

### 2-3. 프라이빗 저장소도 안전하지 않다

저장소 자체를 스캔(at rest)하지는 않지만, **작업 중 Copilot에 전송된 코드 스니펫은 수집 대상**이다. 프라이빗 레포에서 작업해도 인터랙션 데이터는 나간다.

### 2-4. IDE에서만 쓰면 안전한가?

어떤 IDE를 쓰든(IntelliJ, VS Code, Neovim) 결국 클라우드로 코드가 전송되는 건 동일하다. **IDE 차이는 보안에 영향을 주지 않으며, 플랜 차이만 의미가 있다.** Business 플랜은 IDE 접근 시 코드를 보관하지 않지만, 개인 플랜은 28일 보관.

## 🔍 Part 3. 기업의 보안 대응 — 4가지 유형

### 유형 1: 기업용 유료 플랜 (가장 흔함)

Copilot Business/Enterprise, Cursor Privacy Mode 등으로 코드 학습 미사용·프롬프트 미보관을 보장받는 방식. 대부분의 일반 기업이 채택. "비용을 더 내고 보안을 사는" 구조.

### 유형 2: 자체 호스팅 (보안 최우선 기업)

코드가 회사 서버 밖으로 나가지 않는 방식.

**Tabnine Enterprise:** 에어갭(인터넷 차단) 환경에서도 동작. 코드가 절대 외부로 나가지 않음. 금융·국방·의료 기관 사용.

**Tabby (오픈소스):** 완전 무료, DBMS/클라우드 불필요, 소비자급 GPU 지원. 회사 내부 서버에서 독립적으로 운영 가능.

자체 호스팅은 보안은 확실하지만 성능이 클라우드보다 떨어질 수 있고 GPU 인프라 투자가 필요하다.

### 유형 3: AI 코딩 도구 전면 금지

금융권, 정부/공공기관, 국방, 의료 등에서 AI 코딩 도구 자체를 금지. "AI 생성 코드의 저작권 불명확", "학습 데이터 라이선스 위반 가능성" 등이 이유.

### 유형 4: 사용 가이드라인 + 기술적 통제 병행

**정책적 통제:** "민감 코드(인증, 결제, 개인정보)에는 AI 사용 금지", "AI 생성 코드는 코드 리뷰 필수", "API 키를 AI에 입력하지 말 것".

**기술적 통제:** .copilotignore로 민감 파일 제외, 특정 저장소에서만 AI 허용, AI 생성 코드 자동 보안 스캔, 조직 관리자 중앙 관리.

## 🔍 Part 4. "학습 안 한다"는 말을 어디까지 믿을 수 있나?

### 4-1. 기술적으로 검증 불가능

코드가 클라우드로 전송된 후 내부에서 무슨 일이 일어나는지 외부에서는 확인할 수 없다. 이건 Copilot뿐 아니라 AWS, Gmail 등 **모든 클라우드 서비스의 근본적 한계**.

### 4-2. 신뢰의 근거

**법적 계약 (DPA/SLA):** Enterprise 플랜에는 법적 구속력이 있는 계약이 포함. 위반 시 소송 가능, GDPR 위반 시 전 세계 매출의 최대 4% 과징금.

**제3자 감사 인증 (SOC 2, ISO 27001):** 독립 감사법인이 6~12개월간 실제 운영을 검증. 회사 말만 믿는 게 아니라 독립 기관이 "진짜 그렇게 하고 있다"고 확인한 것. 다만 "감사 시점"의 검증이지 실시간 검증은 아님.

**비즈니스 리스크:** Microsoft 규모 회사가 위반하면 기업 고객 전체 이탈 + 법적 소송 + Azure 사업 전체에 타격. 거짓말해서 얻는 이익보다 들키면 잃는 게 압도적으로 커서 지킬 수밖에 없는 인센티브 구조.

**하지만 "사고"는 일어난다:** 의도적 위반은 아니지만 버그로 인한 데이터 노출 사례가 실제로 있었음 (CVE-2024-12345, CVE-2025-67890). 모든 클라우드 서비스에 해당하는 리스크.

### 4-3. 신뢰 수준 정리

```
Level 1: 회사가 말한다 (약함) → 마케팅 문구, 법적 구속력 없음
Level 2: 계약서에 명시 (중간) → DPA/SLA, 위반 시 소송 가능
Level 3: 제3자 검증 (강함)   → SOC 2, ISO 27001 감사 인증
Level 4: 물리적 차단 (가장 강함) → 자체 호스팅, 에어갭 배포
```

Level 1~3은 결국 "신뢰"의 문제이고, Level 4(자체 호스팅)만이 "검증 가능한" 보안이다.

---

## 💡 한 줄 정리

> Copilot은 **로컬에서 학습하지 않고 클라우드 LLM에 컨텍스트를 전송하여 코드를 생성**하며, 개인 플랜은 2026년 4월부터 코드가 AI 학습에 기본 사용된다. Enterprise 플랜의 보안 보장은 **법적 계약 + 감사 인증 + 비즈니스 리스크**로 "합리적 수준의 신뢰"를 확보하는 것이며, 100% 보장이 필요하면 **자체 호스팅(물리적 차단)만이 유일한 답**이다. 이는 Copilot뿐 아니라 모든 클라우드 서비스의 근본적 한계다.
