# 2025-07-11 - 동시성과 병렬성

## 동시성(Concurrency)

1. 개념
   - 동시성은 여러 작업을 동시에 처리하는 것처럼 보이게 하는 것
   - 실제로 동시에 처리되는 것이 아닌 짧은 시간에 여러 작업을 번갈아 실행해 동시에 진행되는 것 처럼 느끼게 함
   - 주로 단일 코어(Single-Core) CPU 환경에서 시분할 기법과 컨텍스트 스위칭을 활용
2. 목적
   - 작업 관리: 여러 작업을 효율적으로 관리하고, 대기 시간을 줄여 전반적인 응답성을 높임
   - 자원 활용: 단일 CPU 코어를 여러 작업이 공유하여 유휴시간 없이 최대한 활용

## 병렬성(Parallelism)

1. 개념
   - 병렬성은 여러 작업을 실제로 동시에 처리하는 것
   - 멀티 코어 CPU, 다중 CPU, GPU 등으로 동시에 여러 작업을 독립적으로 실행
   - 각 처리장치가 할당된 작업을 동시에 수행해 실제로 시간 단축
2. 목적
    - 성능 향상: 대규모 계산이나 처리 집약적인 작업을 여러 처리 장치에 분산해 총 작업 시간 줄임
    - 처리량 증가: 주어진 시간 내 더많은 작업 완료

## 구분 이유

- 설계 관점: 동시성 문제는 자원을 공유하는 여러 작업 간의 순서, 동기화 데드락 등 설계 및 관리의 문제고, 병렬성 문제는 여러 코어에 효율적으로 분배하고 실행하는 문제
- 성능 최적화: 동시성이 중요한 것과 병렬성이 중요한 것에 대한 명확한 구분 필요

## 요약

| 구분 | 동시성(Concurrency) | 병렬성(Parallelism)|
|---|---|---|
| 개념 | 여러 작업을 동시에 진행되는 것처럼 보이게 관리| 여러 작업을 실제 동시 진행 |
| 목표 | 작업 관리, 응답성 향상, 유휴시간 최소화 | 처리량 증가, 작업 완료 시간 단축 |
| 실행 방식 | 단일 코어에서 컨텍스트 스위칭 번갈아 실행 | 멀티 코어/ 다중 CPU에서 각 코어가 동시 실행 |
| 필요 자원 | 단일 처리 장치(CPU 코어) | 다중 처리 장치 (멀티 코어 CPU, 멀티 CPU) |
